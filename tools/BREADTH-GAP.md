Auto-generated by `node tools/breadth-gap.js > tools/BREADTH-GAP.md`. Read-only

Regenerate any time after a catalog change; do not hand-edit this file.

**111 frames** x **9 checked categories** = 999 (frame, category) pairs evaluated.

## Method

For each (frame, category) pair, every candidate part in that category is
placed alone with the frame into `checkBuild({frame, <slot>: candidate})` -
no other slot is filled. With only two slots populated, the only rules that
can fire are ones comparing those two slots directly, so every error here is
attributable to the frame<->candidate pair specifically (see src/compat.js's
`checkBuild`). **Compatible = zero errors.** Warnings (e.g. "exceeds rated
max", "needs a shim") still leave the part usable per this project's own
error/warning split, so they never subtract from the compatible count - a
separate warned-count is reported alongside.

A pair is **flagged** when 0 or 1 distinct brands remain compatible.

## Excluded categories

These have no checkBuild rule that compares them directly to a *frame* field
(the constraint, if any, runs through the fork or another part instead, or the
rule is a warning, or there is no rule at all) - a frame swap alone can never
collapse them, so they are out of scope for this report:

| Category | cat value(s) | Why excluded |
|---|---|---|
| Front brake | `brake` | rule 8 checks it against the FORK's brakeMount, never the frame - a fork-choice axis, not a frame one. |
| Front rotor | `rotor` | rule 10's frame-adjacent-ish check is fork-side (front-rotor-max) and is a WARNING, not an error; rule 9's mount check is vs the front wheel, not the frame. |
| Rear rotor | `rotor` | rule 10 (rear-rotor-max) is a WARNING, not an error - a rotor "too big for the frame" still fits, it is just flagged for review; rule 9's mount check is vs the rear wheel, not the frame. |
| Crankset | `crankset` | rule 7 (bb-advisory) is an INFO note only - it never blocks a build, so no crankset can ever collapse against a frame. |
| Shifter / Cassette / Chain | `shifter, cassette, chain` | no checkBuild rule compares any of these directly to a frame field. |
| Handlebar / Stem / Grips / Saddle / Pedals | `handlebar, stem, grips, saddle, pedal` | no checkBuild rule compares any of these to a frame field at all - frame choice never constrains them. |
